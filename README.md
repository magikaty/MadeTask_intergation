Техническое задание: интеграция внутренней CRM с внешним сервисом MadeTask

**Описание проекта**

Проект направлен на создание интеграции между внутренней CRM-системой компании и внешним сервисом MadeTask.
Интеграция решает задачи централизации учёта исполнителей, автоматизации выплат и оптимизации бизнес-процессов, связанных с постановкой и оплатой задач.

Цель и задачи

**Цель проекта:** создание надёжной интеграции, обеспечивающей автоматизированное и безопасное взаимодействие между CRM и MadeTask без ручного участия менеджеров в операциях по задачам и выплатам.

**Ключевые задачи системного анализа:**

* исследование текущих бизнес-процессов (“as is”) и выявление узких мест;
* разработка целевой модели процессов (“to be”);
* формализация функциональных и нефункциональных требований;
* проектирование архитектуры интеграционного решения;
* описание API-взаимодействий и ролей пользователей.

Бизнес-контекст

**Проблемы исходной системы**:
* задачи и коммуникации ведутся через мессенджеры, отсутствует централизованный контроль;
* ручное создание задач и изменение статусов в CRM;
* неавтоматизированная система выплат исполнителям;
* высокий риск ошибок и потери данных из-за дублирования ручного труда.

**Целевое решение**:
* интеграция CRM и MadeTask по REST API;
* централизованный учёт исполнителей и задач;
* автоматизация выплат;
* контроль статусов и уведомлений в единой системе.

Функциональные требования
* синхронизация данных об исполнителях и задачах между CRM и MadeTask;
* создание и редактирование задач;
* изменение статусов задач с автоматическим запуском выплат;
* формирование логов и уведомлений об ошибках;
* ролевая модель доступа (администратор, менеджер, исполнитель).

Архитектура решения

**Основные компоненты:**
* внутренняя CRM-система заказчика;
* интеграционный модуль (middleware);
* API внешнего сервиса MadeTask.

**Технологии и протоколы:**
REST API (JSON), OAuth2, HTTPS/TLS 1.2+

**Серверные требования:** ≥ 2 CPU, ≥ 4 GB RAM.

Нефункциональные требования

| Категория              | Ключевые параметры                                                     |
| ---------------------- | ---------------------------------------------------------------------- |
| **Производительность** | отклик ≤ 3 сек, до 1000 выплат ≤ 5 мин                                 |
| **Надёжность**         | восстановление без потери данных, время недоступности ≤ 1 ч/мес        |
| **Безопасность**       | HTTPS/TLS 1.2+, разграничение ролей, защита персональных данных        |
| **Масштабируемость**   | до 10 000 исполнителей без переработки архитектуры                     |
| **Совместимость**      | REST API, JSON, соответствие бизнес-логике текущей CRM                 |
| **Сопровождение**      | логирование операций, уведомления об ошибках, мониторинг синхронизаций |

Тестирование и приёмка

Проверка успешности интеграции включает:
* тестирование функционала (создание, обновление, оплата задач);
* нагрузочные тесты;
* аудит безопасности и отказоустойчивости;
* проверку корректности бизнес-логики.

**Приложения**

* As-Is схема взаимодействия;
* Use Case диаграммы;
* UML-диаграммы последовательностей;
* BPMN-диаграммы процессов регистрации и постановки задач;
* Описание API MadeTask (создание исполнителя, задачи, изменение статуса, оплата).
